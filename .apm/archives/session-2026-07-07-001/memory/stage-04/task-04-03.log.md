# Task 4.3 - Geração do Pedigree em PDF

## Summary
Certificado de pedigree em PDF implementado com `@react-pdf/renderer`, reaproveitando `montarArvoreGenealogica` (Task 4.1) como única fonte de ancestralidade, e ligado ao link "Gerar pedigree" já existente na Ficha da Ave.

## Details
1. `@react-pdf/renderer` instalado — renderização React→PDF em JavaScript puro (sem navegador headless), conforme instruído.
2. `lib/pedigree/verificacao.ts`: `gerarCodigoVerificacao(anilha, tenantNome)` — função pura, formato `PDG-{4 últimos dígitos da anilha}-{iniciais das duas últimas palavras do nome do criatório}` (ex: `PDG-0451-SV` para "Aviário Serra Verde" — as duas últimas palavras evitam capturar um prefixo genérico como "Aviário"/"Sítio"). Puramente cosmético, sem endpoint de consulta, conforme instruído.
3. `lib/pedigree/service.ts`: `montarDadosPedigree(aveId, tenantId, responsavelNome)` busca `getAve` (dados de exibição da ave), `montarArvoreGenealogica` (Task 4.1 — única fonte de dados de ancestralidade, sem nova consulta) e o `Tenant` (nome/logo) em paralelo, e monta o objeto `DadosPedigree` consumido pelo template.
4. `lib/pedigree/PedigreeDocument.tsx`: template `@react-pdf/renderer` reproduzindo `design/11 Exportar Pedigree.dc.html` — cabeçalho (logo/nome do criatório, título "Certificado de Pedigree", data de emissão por extenso), resumo da ave (foto ou placeholder, nome, espécie/mutação/sexo, anilha/nascimento/origem), árvore (geração 0 destacada, pais, avós com os rótulos de ausência da Task 4.1), rodapé (responsável + selo + código de verificação). Paleta de cores fiel ao design; fontes substituídas pelas famílias embutidas do `@react-pdf/renderer` (ver Compatibility Concerns).
5. `app/plantel/[id]/pedigree/route.ts`: Route Handler que gera o PDF (`renderToBuffer`) e retorna como download (`Content-Disposition: attachment`), ligando o link "Gerar pedigree" já existente na Ficha da Ave (criado na Task 2.4, apontava para essa rota sem implementação até agora).
6. **Achado importante**: o nome completo do responsável, coletado no formulário de cadastro (Task 1.3, campo "Nome completo"), nunca é persistido em lugar nenhum — nem no Supabase Auth (`user_metadata`) nem no `Tenant`. Como o certificado precisa desse dado, usei o e-mail do usuário autenticado como alternativa disponível, documentado no código e aqui. Recomendo uma Task futura para persistir esse nome (ex: em `user_metadata` no `signUpAction`, ou um campo dedicado no `Tenant`).
7. Testes automatizados: `lib/pedigree/verificacao.test.ts` (3 casos do gerador de código), `lib/pedigree/PedigreeDocument.test.tsx` (estrutural, conforme pedido pela Task — extrai todo o texto da árvore de elementos React retornada pelo componente, sem depender do binário do PDF, confirmando a presença de todas as seções: ave, pais, avós com rótulos de ausência, criatório, responsável, código de verificação), `lib/pedigree/service.test.ts` (integração: monta os dados contra o banco real e gera um PDF de verdade, confirmando o buffer não vazio e a assinatura de arquivo `%PDF`).
8. Ao rodar a suíte completa, o teste de integração desta Task ocasionalmente esbarrou em `EMAXCONNSESSION` (limite de conexões do pooler do Supabase) sob a concorrência de 18 arquivos de teste rodando em paralelo — confirmado como transitório (passou de forma consistente ao reexecutar). Mesma categoria de contenção de recursos já observada na Task 3.4 (que motivou aumentar o timeout do Vitest); não é uma falha de lógica.

## Compatibility Concerns
As fontes do design (`Source Serif 4`, `Manrope`, `Space Grotesk`) foram substituídas pelas famílias embutidas do `@react-pdf/renderer` (Times/Helvetica/Courier, nos pesos Bold/Italic disponíveis) para aproximar a hierarquia visual (serif para títulos/nomes, sans para corpo/rótulos, monospace para anilha/código) sem precisar baixar e empacotar arquivos de fonte de terceiros — não tive uma fonte confiável e já aprovada para essas famílias específicas no formato exigido pelo React-PDF (TTF/OTF), e evitei inventar/baixar arquivos de fonte de origem não verificada. Se a fidelidade tipográfica exata for necessária, uma Task futura pode adicionar os arquivos de fonte corretos (fornecidos pelo usuário ou de uma fonte licenciada verificada) e registrá-los via `Font.register`.

## Important Findings
O nome completo do responsável (coletado no cadastro, Task 1.3) nunca é persistido — nem no Supabase Auth nem no `Tenant`. O certificado de pedigree usa o e-mail do usuário como alternativa até essa lacuna ser resolvida por uma Task dedicada.
