# Task 1.1 - Inicialização do projeto e infraestrutura base

## Summary
Scaffold Next.js (TypeScript, App Router, Tailwind) criado na raiz do repositório e Prisma configurado e conectado ao banco Postgres do Supabase via connection pooler. Commit inicial registrado na branch `feat/setup-multitenancy-onboarding`.

## Details
1. `create-next-app` recusou rodar diretamente na raiz por já haver arquivos (`.apm/`, `CLAUDE.md`, `design/`). Contornado gerando o scaffold em um diretório temporário e mesclando via `rsync` (excluindo `.git`), preservando os arquivos já existentes do projeto.
2. O scaffold trouxe um `AGENTS.md` e sobrescreveu o `CLAUDE.md` do projeto com um stub `@AGENTS.md` gerado pelo Next.js. Detectado imediatamente após a mesclagem; `AGENTS.md` removido e o `CLAUDE.md` original (regras do projeto) restaurado a partir do conteúdo já lido nesta sessão.
3. Renomeado `package.json` name de `app` para `plantel-board`.
4. Instalado Prisma + `@prisma/client`, rodado `prisma init --datasource-provider postgresql`, instalado `dotenv` (exigido por `prisma.config.ts` gerado).
5. Primeira tentativa de conexão com a connection string direta do Supabase (`db.<ref>.supabase.co:5432`) falhou com `P1001`. Diagnóstico via `dig`: o host só resolve para IPv6 (`AAAA`), sem registro `A`, e a rede local não tem conectividade IPv6. Reportado ao usuário, que forneceu a string do connection pooler (Supavisor, `aws-1-us-west-2.pooler.supabase.com:5432`), que resolve por IPv4 e funcionou.
6. Senha do banco continha o caractere `@`, exigindo URL-encoding (`%40`) na connection string.
7. `next.config.ts` ajustado com `turbopack.root` para eliminar aviso de lockfile duplicado detectado (havia um `package-lock.json` não relacionado em `~/`).
8. `.env` confirmado como ignorado pelo git (`.gitignore` já continha `.env*`) antes do commit.

## Important Findings
1. A conexão direta do Supabase (porta 5432 no host `db.<ref>.supabase.co`) não é alcançável nesta rede por exigir IPv6. Qualquer configuração futura de `DATABASE_URL` (local dev, CI, deploy) precisa usar o connection pooler do Supabase (Supavisor) ou uma rede com suporte IPv6. Isso pode afetar a Task 1.2 e configurações de ambiente futuras.
2. O `create-next-app` sobrescreveu `CLAUDE.md` com um arquivo gerado pelo próprio Next.js (stub `@AGENTS.md`). Foi corrigido nesta Task, mas é um risco a observar em qualquer scaffold futuro rodado na raiz do projeto: sempre verificar se `CLAUDE.md` permanece intacto após ferramentas de scaffolding.
