# Task 1.2 - UI de adicionar espécie — inline e em Configurações

## Summary
Duas superfícies de interface implementadas para adicionar uma espécie nova — inline no fluxo de Novo Cadastro de Ave, e em uma seção de gestão do catálogo em Configurações — ambas chamando o endpoint da Task 1.1 sem duplicar a lógica de normalização no cliente.

## Details
1. `lib/especies/client.ts` (novo): `criarEspecie(nome)` — único ponto de chamada ao `POST /api/especies`, reaproveitado pelas duas superfícies (evita duplicar a lógica de fetch/erro em dois componentes).
2. `app/plantel/novo/NovoCadastroForm.tsx`: adicionado o link "Não encontrou sua espécie? Adicione aqui" abaixo do select de Espécie. Ao clicar, revela um campo de texto + botões Adicionar/Cancelar (mesmo padrão visual dos botões de edição inline já usados em Configurações, Task 5.1). Ao adicionar com sucesso, a nova espécie é inserida na lista local (`listaEspecies`, elevada de prop para estado, permitindo atualização sem reload) já ordenada alfabeticamente, e automaticamente selecionada no dropdown.
3. `app/configuracoes/ConfiguracoesView.tsx`: nova seção `EspeciesSection`, reaproveitando exatamente o padrão visual de `CriatorioSection` (cartão com lista + ação de adicionar) — sem introduzir um componente ou estilo novo, conforme instruído (nenhum mockup novo foi produzido para esta funcionalidade).
4. `app/plantel/novo/page.tsx` e `app/configuracoes/page.tsx`: atualizados para buscar as espécies via `listarEspecies()` (Task 1.1) em vez de uma consulta Prisma direta duplicada — `configuracoes/page.tsx` agora busca espécies em paralelo com a preferência de alertas via `Promise.all`.
5. Verificado visualmente via rota temporária (`app/visualtestespecies`, removida antes do commit) com dados mocados: layout de ambas as superfícies confere com os tokens do design system existente; confirmado via inspeção da árvore de acessibilidade que a seção de Configurações lista as três espécies em ordem alfabética com o botão "Adicionar espécie"; o fluxo de submissão do formulário inline foi exercitado de ponta a ponta contra o servidor real (sem sessão autenticada válida no mock, retornou corretamente "Onboarding não concluído para este usuário" — confirma que a chamada chega ao endpoint real e que o tratamento de erro no cliente funciona).
