# Task 1.1 - API de criação de espécie com normalização e checagem de duplicata

## Summary
`POST /api/especies` implementado com normalização de capitalização e reaproveitamento de entrada existente equivalente (dedup case-insensitive); lógica exposta em `lib/especies/service.ts` para reaproveitamento futuro pela importação em lote (Task 2.3); listagem mantida em ordem alfabética.

## Details
1. Validado o estado real de `app/api/especies/route.ts` antes de implementar: já existia um `GET` simples (Fase 1) já ordenando por `nome: "asc"` — sem `POST`.
2. `lib/especies/normalizar.ts`: `normalizarNomeEspecie(nome)` — capitaliza a primeira letra de cada palavra separada por espaço, colapsando espaços extras (`"canário   belga"` → `"Canário Belga"`).
3. `lib/especies/schema.ts`: schema Zod `createEspecieSchema` (`nome` obrigatório, não vazio), seguindo o mesmo padrão de `lib/aves/schema.ts`.
4. `lib/especies/service.ts`: `criarOuReaproveitarEspecie(input)` — normaliza o nome, busca todas as espécies existentes e compara case-insensitive em JS (evitando depender do suporte a `mode: "insensitive"` do Prisma, que exigiria confirmação de compatibilidade com o provider Postgres sem extensão `citext`); reaproveita a entrada existente se equivalente, senão cria. Retorna `{ especie, criada }` para o chamador decidir o código HTTP (200 reaproveitada / 201 criada). Também extraí `listarEspecies()` (ordenação alfabética) para a mesma camada de serviço, mantendo a rota fina.
5. Endpoint `POST` no route.ts usa `criarOuReaproveitarEspecie`, traduzindo `ZodError` para 400.
6. **Achado corrigido durante a validação**: `vitest.config.ts` não excluía o diretório `e2e/` (suíte Playwright, Task 5.3), fazendo o Vitest tentar executar `e2e/fluxo-completo.spec.ts` como se fosse um teste próprio e falhar na API `test.describe.configure` (específica do Playwright). Adicionado `exclude: ["**/node_modules/**", "e2e/**"]`.
7. Testes automatizados: `lib/especies/normalizar.test.ts` (capitalização, maiúsculas, espaços extras, palavra única), `lib/especies/service.test.ts` (integração: criação com normalização, reaproveitamento de duplicata por grafia/caixa diferente confirmando contagem única no banco, rejeição de nome vazio, e ordenação alfabética da listagem com nomes reais criados no teste).
