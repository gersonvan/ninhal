# Task 4.4 - Tela Exportar Pedigree

## Summary
Tela de visualização/exportação de pedigree implementada em `/plantel/[id]/pedigree`, reproduzindo `design/11 Exportar Pedigree.dc.html` e reaproveitando `montarDadosPedigree` (Task 4.3) como única fonte de dados. "Exportar PDF" liga à geração de PDF já existente; "Imagem" e "Compartilhar" ficam como ações secundárias não funcionais, conforme instruído pela Task.

## Details
1. Conflito de rota resolvido: `app/plantel/[id]/pedigree/route.ts` (geração de PDF, Task 4.3) foi movido via `git mv` para `app/plantel/[id]/pedigree/download/route.ts`, liberando o path principal `/plantel/[id]/pedigree` para a nova `page.tsx` — Next.js não permite `page.tsx` e `route.ts` no mesmo segmento.
2. `lib/pedigree/responsavel.ts`: extraído `resolverNomeResponsavel(user)` como helper único para a lógica de "nome do responsável = e-mail autenticado como alternativa" (achado da Task 4.3), agora compartilhado entre a rota de download e a nova página, evitando duplicação e concentrando em um único lugar o ponto que a Task 4.5 precisará alterar.
3. `app/plantel/[id]/pedigree/page.tsx`: Server Component que valida autenticação/onboarding (mesmo padrão das demais páginas do Plantel), monta os dados via `montarDadosPedigree` dentro de `runWithTenant`, e trata `RegistroNaoEncontradoError` como 404 (`notFound()`). A busca de dados e o tratamento de erro foram extraídos para uma função auxiliar `buscarDadosOu404` para evitar construir JSX dentro de um bloco `try/catch` (regra de lint `react-hooks/error-boundaries`).
4. `app/plantel/[id]/pedigree/ExportarPedigreeView.tsx`: reprodução fiel do certificado do design — cabeçalho (logo/nome do criatório, título "Certificado de Pedigree", data de emissão), resumo da ave, árvore genealógica (geração 0 destacada, cartões de pai/mãe, cartões de avós reaproveitando os rótulos de ausência da Task 4.1), rodapé (responsável, selo "AUTENTICADO", código de verificação). Barra de ações no topo: "Voltar" (link para a Ficha da Ave), "Imagem" e "Compartilhar" (botões sem `onClick`, intencionalmente não funcionais), "Exportar PDF" (link para a rota de download).
5. Conforme instrução explícita da Task, nenhuma integração não solicitada (Web Share API, conversão para imagem) foi adicionada para "Imagem"/"Compartilhar" — o design não especifica destino funcional para essas ações além do PDF.
6. Verificação visual feita via rota temporária `app/visualtestpedigree/page.tsx` com dados mocados (família Amália/Rufus/Alice, incluindo avós com ambos os tipos de rótulo de ausência), comparando cabeçalho, resumo, árvore e rodapé lado a lado com o mockup — removida antes do commit.
